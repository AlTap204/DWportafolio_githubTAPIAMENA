Sobre este proyecto

Este proyecto fue desarrollado durante el curso DW0619 - Diseño Web, aplicando todas las habilidades vistas en clase. A continuación, detallo las principales actividades realizadas:

Comencé construyendo la estructura base de mi sitio web personal: un portafolio académico. Inicialmente implementé una página estática con Flask, organizando las carpetas de templates, static, y archivos HTML como index.html, acerca.html y portafolio.html. En esta estructura incorporé secciones navegables con rutas dinámicas y contenidos personalizados.

Una de las primeras actividades fue crear una página de inicio con una interfaz amigable. Para ello utilicé CSS personalizado, aplicando colores, tipografías agradables y diseño responsivo. Posteriormente, integré una imagen de fondo en el encabezado (cielobackground.webp) y una foto de perfil redonda (mifoto.webp) junto a mi nombre, lo que le dio identidad visual al sitio.

En la sección "Sobre mí", agregué un enlace profesional a mi perfil de LinkedIn, el cual está centrado visualmente con un botón estilizado y funcional. Esta vista replica el diseño institucional de la página de inicio, manteniendo una coherencia estética.

Luego integré una sección de "Sobre el proyecto", donde documenté el desarrollo de todas las actividades del curso. En ella destaqué el uso de JavaScript para interacciones sencillas, como mostrar mensajes mediante funciones personalizadas. También incluí una descripción detallada de prácticas importantes como:

El uso de Nmap para escanear puertos en el servidor local y detectar servicios abiertos. La práctica con Wireshark para capturar paquetes de red y observar diferencias entre los protocolos HTTP y HTTPS. Todo esto lo desarrollé en un entorno local usando Flask y Visual Studio Code en Ubuntu. Cada vista fue estilizada cuidadosamente con CSS para garantizar una experiencia visual limpia, organizada y profesional.

Finalmente, este portafolio fue diseñado no solo como una práctica académica, sino como una base sólida para seguir aprendiendo y publicarlo en el futuro mediante GitHub Pages, como una versión estática que no requiere servidor backend.
